## 1. ЧТО ЗА ЗАДАЧА
Полный **Design Completion Pass**: mobile-first responsive-оболочка, переработка главных экранов и единая визуальная система «как у Motion по ощущению, но на компонентах Needt».

**Зафиксированные решения (не отменять):**
- **Никакого отдельного дерева `/mobile/...`** — мобильные композиции внутри тех же роутов, те же данные/логика.
- **Нативное приложение сейчас НЕ делаем** — качественный responsive **PWA** (те же роуты/API, мобильные композиции, bottom sheets, safe-area, touch-действия, install-иконка). Позже можно обернуть Capacitor / перенести в React Native без смены сервера.
- Motion — **референс по меркам и поведению, не копирование HTML** (его DOM завязан на их код).
- **Мобилка = собственный слой, НЕ ужатый десктоп.** Главный мобильный референс — **Tiimo** (см. `design-refs/screens/README.md`, раздел MOBILE): нижняя таб-панель (To-do/Today/Focus/Me), чистые экраны Today/Focus/Settings. Берём СТРУКТУРУ Tiimo, рендерим в тёмной тёплой системе Needt (Tiimo светлый — тему не копировать).
- **Паритет фич на мобилке НЕ обязателен** (решение заказчика). Приоритет: Today, Focus, To-do/Tasks, Settings довести до вида/рабочего состояния; Calendar week-grid / Boards / Space — упростить под телефон или «best on desktop».

## 2. РЕФЕРЕНСЫ MOTION (промерено в Chrome — использовать как ориентир)
- Основной фон `rgb(26,29,30)`; панели отличаются на 3–6 уровней яркости; обычные границы `rgb(43,47,49)`; контролы высотой **25–30px**, радиус **6px**.
- **«Сглаживание» = очень близкие поверхности + тонкие границы + стабильная типографика + минимальный контраст соседних слоёв. НЕ blur/glow.**
- **Agenda/Today** — простой документ ~**800px**, текст **15px/24px**, заголовки **20px**, БЕЗ карточки вокруг каждой задачи.
- **Task modal** — **1018px**, правая панель ровно **320px**, спокойные строковые поля.
- **Date picker** — **443px** (календарь 262px + быстрые даты 180px: Today / Tomorrow / Next week / In 2 weeks / Next month).
- **Settings** — один заголовок страницы, дальше только названия секций (без повторного описания категории).
- **Inbox** — toolbar **65px**, компактные строки **40px**.

## 4. ЧТО ОСТАЛОСЬ (по приоритетам заказчика)

### P1 — основные экраны
- **Calendar:** читаемость событий на узких колонках; единый визуальный язык событий и задач; лучше выделять выбранное событие; доработать quick-create и detail popover; проверить Month/Day/Week/Timeline отдельно. **Quick-create — сделать как у Motion и добавить больше разных вариантов** (заметка: один баг виден только в локальной dev-сборке, на сайте нет).
- **Today:** desktop сейчас слишком узкий → **немного увеличить ширину контента (как Motion, ~800px agenda)**; вертикальный ритм; аккуратные hover-действия на задачах; продумать пустой день / завершённые / перегруженный день; крупный заголовок оставить.
- **Create/Edit Task:** единая сетка полей; явное **Saving… / Saved**; предупреждать при закрытии с несохранёнными; лучше мобильный bottom-sheet; Advanced settings менее перегружены.
- **Settings:** пройти ВСЕ вкладки; **убрать второе название/описание категории**; одинаковые `SettingsSection`/строки/переключатели/Connect-кнопки; **синие кнопки скопировать по стилю с Motion (сейчас везде разные — свести к одному стилю)**; унифицировать `Connected / Not configured / Unavailable`; кликабельные «coming soon» — сделать явно недоступными (спокойный серый, профессионально); visual regression на каждую вкладку. **Date picker в настройках — как у Motion (скрин 6), единый компонент.**

### P2 — остальные разделы
- **Boards:** пустое состояние, колонки, drag-feedback, открытие карточки.
- **Space:** верхняя панель спокойнее, выбранная задача, читаемость подписей; мобильную заглушку можно оставить.
- **Focus:** ЗАВЕРШИТЬ Opal-style ре-райт (стиль UI/UX как у Needt, расположение как у Opal). Сейчас начато, доделать.
- **Mail:** список писем / reading pane / подключение аккаунта / пустые состояния в стиле Needt; ориентир — **Inbox Motion** (toolbar 65px, строки 40px).
- **AI Chat:** убрать отдельные цвета/стили, унифицировать composer, tool calls, загрузку.

### P3 — ощущение качества
- Анимации **150–250ms, без glow**; плавное появление popover/modal/bottom-sheet; press-feedback на мобильных кнопках; **skeleton вместо пустого экрана**; полная клавиатурная навигация; touch-зоны **≥44px**; контраст вторичного серого текста; **`prefers-reduced-motion`**.
- **Главное раздражение заказчика:** элементы местами «рябят» / слишком резкие, по-разному на разных экранах — добавить **сглаживание как у Motion** (см. §2: близкие поверхности + тонкие границы, а не резкие контрасты; проверить `antialiased`, субпиксельные границы, целочисленные размеры/позиции).
- Осветление иконок слева уже убрано — проверить, что нигде не осталось.
